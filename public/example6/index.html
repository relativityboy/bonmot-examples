Example 6. Switching Models in Child-Views

Change a child Model and watch the Child-View update.

This example is a little more interesting. Our example code has a list of four Address.Models, each with a corresponding button. Clicking on the button will set the corresponding Address.Model on the Person.Model.

Click the buttons to switch addresses. Because the address models are persistent and edits you make to an address will be preserved.

In the code:

  1. There are no changes to Address files
  2. Person.View is the only JS that's changed at all. A list of example Address.Models exampleAddresses has been added, and a controller ctrlSetAddress to handle switching
  3. person.hbs has a static list of buttons for switching addresses.

In the console:

  1. person.$ctrl.setAddress
  2. person.exampleAddresses
  3. person.ctrlSetAddress
  4. person.hbs()
  5. person.model.toJSON()

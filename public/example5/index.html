Example 5. Automagic Child-Views

Create a simple child-view.

This is a fairly uncomplicated child-view. We've added an address below to the person. There's a corresponding /address app directory with our example, with CSS, HTML, and Javascript files. Not much different from what you saw in Example1.

This user interface doesn't 'do' anything new other than display the address view and model data. There's more interactivity in the next example.

In the code: Our Child-View magic happens in the Person app. You'll see two things.

  1. In Person.Model we've added a _set.address function (courtesy of DW-Backbone to make sure that any JSON being set on 'address' gets turned into an Address.Model instance.
  2. In Person.View we've added atrViews.address and said to use the Address.View to create a view for the address on the Person.Model that our Person.View is handling at any particular moment

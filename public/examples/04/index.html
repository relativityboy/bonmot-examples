Example 4. A touch of color.

Modularizing CSS with the 'unique' attribute.

In this example we do something trivial, but really nice. We've added an attribute 'unique' to our View. It's intended to be unique across all the View Types you'll load. What's really neat about this string is that it's injected into the root element of your view as a css class (dots are transformed into dashes).

This helps you more easily create View specific css and know it'll be applied where it belongs.

In the code: There's a new css file in the person directory, and two more lines in our person.js file. One to import the css, and another for the 'unique' attribute.
(you could just as easily include the css via direct link in your page)

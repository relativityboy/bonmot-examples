Example 9 Collections of Models with Child-Views.

Sorting, Adding, and Removing items from lists.

In this example we move from a simple list to a table that sorts when you click a column header. See Person.View.ctrlSortBy. Clicking on a row in the table will set that model on the Address Detail view.

Also we have Person.View.ctrlNewAddress and Person.View.ctrlDeleteAddress for creating and deleting models.

*BonMot does not yet 'clear' the w-atr-* fields when a model is unset and without a model to replace it. For this example, when you hit delete an address model, the address ItemView will disappear from the list on the left, but the input fields still display the deleted model's information.
Clearning this up (no pun intended) is the next feature on the docket. This notice will disappear when that's been implemented.
Meanwhile, the terrible hack of setting an empty model on the view easily cures the symptoms.

Example 1. Two way data binding.

If you edit the input fields, you'll see the large font name update dynamically.

Not a big deal, but the only 'magic' in the html (example1/person/person.hbs) are nicely formatted css classes, w-atr-displayName, w-atr-firstName, w-atr-lastName.

The View (example1/person/person.js) has exactly two lines of code. uiBindings:['firstName','lastName', 'displayName'] tells BonMot which model attributes to bind. hbs:tplPerson says "use this template for my html".

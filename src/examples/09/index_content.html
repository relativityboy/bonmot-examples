Auto-magic lists of Child Views

This example has exactly the same functionality as example 8. The difference is in how it's implemented.

In 8, the child view, Address.ItemView recieved a click event, and then set its model on the parent-view's model.editingAddress. From an event binding perspective it was nice, and the Javascript code was very clear, but setting items on the parent-view's model is not a great practice. As a side note, the item view html also wound up being more complex, as the entire view needed to be a control

In 8a, we move the control function into the parent-view, and instead take advantage of BonMot's built-in features to determine which model to pluck from the list and set in the editingAddress property.

These two approaches show off different aspects of BonMot's functionality. I'll be using the pattern in 8a for the remaining examples. What you do is up to you.

Choose wisely.
